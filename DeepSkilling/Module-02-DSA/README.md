## Analysis of Algorithms

### Why Data Structures and Algorithms?

Data Structures organize data efficiently, while Algorithms define the steps required to solve a problem.

### Complexity Analysis

#### Time Complexity

Measures the execution time of an algorithm as input size increases.

#### Space Complexity

Measures the amount of memory required by an algorithm.

### Asymptotic Notations

* **Big O (O)** – Worst Case
* **Omega (Ω)** – Best Case
* **Theta (Θ)** – Average Case

### Complexity Cases

* Best Case
* Average Case
* Worst Case

---

## Arrays

Arrays store elements in contiguous memory locations.

### Operations

* Traversal
* Searching
* Insertion
* Deletion

### Advantages

* Fast access using index
* Easy implementation

### When to Use Arrays

* Fixed-size collections
* Frequent element access by index

---

## Linked Lists

Linked Lists store elements using nodes connected through references.

### Types of Linked Lists

#### Singly Linked List

Each node points to the next node.

#### Circular Singly Linked List

Last node points back to the first node.

#### Doubly Linked List

Each node contains pointers to both next and previous nodes.

#### Circular Doubly Linked List

Both next and previous links form a circular structure.

### Operations

* Insert
* Delete
* Search
* Traverse

---

## Searching Algorithms

### Linear Search

Searches elements one by one until the target is found.

**Time Complexity:** O(n)

### Binary Search

Searches by repeatedly dividing a sorted array into halves.

**Time Complexity:** O(log n)

---

## Sorting Algorithms

### Bubble Sort

Repeatedly swaps adjacent elements.

**Time Complexity:** O(n²)

### Insertion Sort

Places elements in their correct position one by one.

**Time Complexity:** O(n²)

### Heap Sort

Uses a Heap data structure for sorting.

**Time Complexity:** O(n log n)

### Quick Sort

Uses Divide and Conquer with a pivot element.

**Average Time Complexity:** O(n log n)

### Merge Sort

Divides data into smaller parts and merges them in sorted order.

**Time Complexity:** O(n log n)
